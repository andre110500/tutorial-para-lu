🚀 Paso 4: Subir a GitHub y entregar

GitHub es donde vas a subir tu proyecto para que el profesor lo vea.

1️⃣ Crear una cuenta en GitHub

  1. Andá a github.com
  2. Hacé clic en "Sign up"
  3. Poné tu email, una contraseña y un nombre de usuario
  4. Verificá tu email

2️⃣ Crear un repositorio

  1. En GitHub, hacé clic en el botón "+" (arriba a la derecha) → "New repository"
  2. Poné como nombre: proyecto-web-lu
  3. Seleccioná "Public" (para que el profesor pueda verlo)
  4. No marques nada más, hacé clic en "Create repository"
📸 En la página que aparece después, vas a ver una sección que dice "Upload an existing file" — hace clic ahí.

3️⃣ Subir los archivos

  1. En la página de "Upload files", arrastrá toda la carpeta de tu proyecto
  2. Abajo escribí un mensaje como: "Primera entrega"
  3. Hacé clic en "Commit changes"
⚠️ Importante: Asegurate de subir TODA la carpeta, no solo los archivos sueltos. La estructura tiene que ser: proyecto-web-lu/index.html, proyecto-web-lu/css/style.css, etc.

4️⃣ Editar el README

El archivo README.md es la carta de presentación de tu proyecto. Tiene que incluir:

  • ✅ Nombre del proyecto
  • ✅ Descripción breve
  • ✅ Integrantes del grupo (nombre completo y rol)
  • ✅ Instrucciones para visualizar el proyecto

5️⃣ Checklist final antes de entregar

Marquen todo lo que ya tienen listo:

  • 3 páginas HTML vinculadas entre sí
  • CSS externo con estilos coherentes
  • Cabecera con menú de navegación
  • Pie de página
  • Texto, imágenes y enlaces relacionados con la temática
  • Una tabla (precios, cronograma, etc.)
  • Un formulario (contacto, registro, etc.)
  • Diseño responsivo (se ve bien en celular)
  • Animación o transición
  • README completo en GitHub

📦 ¿No querés escribir nada? Usá el código completo

En la página "Código completo" tenés todo el proyecto listo. Solo tenés que copiar, descargar y subir a GitHub.
